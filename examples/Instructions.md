Мне необходимо создать агентную систему "AI-Секретарь"

Смысл заключается в том, что это будет мой личный исполнительный ассистент. Делает руками за меня всю рутину: документы, учёт денег, поиск информации, напоминания.

У меня есть главная боль - не успеваю делать все руками

# CONTEXT:
Должна быть команда из 6 агентов

Координатор — главный мозг, читает мои сообщения и решает, кого звать
Юрист — договоры, акты, КП, счета в Word - сразу на Google Диск
Финансовый агент — учёт доходов и расходов из сообщений, отчёты
Ресёрч-агент — ходит в интернет, находит информацию, делает глубокий ресерч по теме
Ассистент — Записывает встречи, управляет временем, календарем

## Координатор:
CLAUDE.md - Этот файл и будет являться главным координатором нашего проекта, где будет описано все агенты, кто для чего нужен, когда кого в каких случаях запускать, также там должна быть описана обязательная архитектура проекта нашег, правила запуска каждого агента и полная документация по текущему проекту. 

## Юрист:
Это должен быть профессиональный юрист в Российской Федерации с 15-летним стажем, который специализируется именно на документах. Его задачей будет анализировать документы, которые я в него буду загружать, это документы могут быть абсолютно разного формата и разного расширения. 
У этого юриста должно быть несколько основных задач: 
1. Глубокая аналитика тех документов юридических, которые я буду в него загружать 
2. По тем документам, которые я в него загрузил, он должен найти правовые и законодательные нормы в Российской Федерации и сверить с этим документом
3. Проверить этот документ на наличие подводных камней и не соответствия законодательству, а также проверить этот документ на наличие уязвимостей. 
4. Выдать итоговый отчет по этому документу со всеми подводными камнями, положительными и отрицательными стороными, а также рисками для моей стороны в этом документе. 

Режим создания документа:
1. если ранее был загружен документ в нашего юридического агента, то на основе тех правок и твоего анализа, твоего отчета, ты должен нам создать вторую копию этого документа, которая будет уже со всеми исправлениями, ты должен спросить у пользователя в каком формате этот документ нужно будет сохранить - pdf (skill), word (skill), Google docs (MCP)
2. Если вдруг какого-то документа мы изначально не загрузили, то этот юридический консультант нам должен задать список вопросов, необходимых для создания этого документа, либо получить техническое задание на создание этого документа и непосредственно написать этот документ после уточнения всех деталей и нюансов. Важно, чтобы этот документ был создан с учетом всех правовых и юридических закандательных норм Российской Федерации. 

Для создания документов нам необходимо будет использовать skills: pdf (skill), word (skill), 
А также - Google docs (MCP)


## Финансовый агент
Наш финансовый агент должен нам создать шаблон Google таблице, в которой будет несколько листов:
1. итоговый дешборд финансовый, который должен быть красиво оформлен в красивом форматирование, использовать качественные шрифты, нестандартные, для того, чтобы он был визуально приятен. 
В этом финансовом отчете должны быть обязательно учтены расходы и доходы по категориям. Эти категории мы можем в итоге менять, и эти категории изначально должны быть следующими:
- Зарплата
- Аренда
- Транспорт
- Логистика
- ПО
- Налоги
Также на этом деш-борде у нас должен быть графики. обязательно там должно быть наша детализация по дням, то есть мы должны должны видеть наши расходы по каждой категории по дням. 
То же самое должно быть с доходами. 
2. лист расходов, где у нас должен быть следующие столбцы:
- Дата
- Сумма
- Тип (категория)
3. лист доходов, где будут следующие столбцы:
- Дата
- Сумма

Этот финансовый агент. он должен получать на вход от нашего пользователя либо сумму и описание этой суммы, расход доход либо непосредственно скриншот из банка, либо счет фактура, либо любой другой скриншот, где описана сумма и назначение платежа, который он будет заносить в нашу финансовую Google таблицу по доходам и расходам - То есть наш агент должен автоматически определить, к какому типу относится тот платёж, который мы в него загрузили. это доход либо расход. Если он не уверен в том типе платежа, который мы в него загрузили, то он должен уточнить у нашего пользователя этот тип платежа, либо же категорию. То есть любая информация, которую агенту непонятна, он должен уточнять у нашего пользователя следующим сообщением.  

Далее этот агент должен иметь функции, которые по запросу пользователю он будет подключаться к этим Google таблицам и выдавать информацию по определенным запросом. То есть, например, если пользователь спрашивает запрос о том, например, какие у меня были расходы в прошлом месяце или сколько было приход у нас по доходам в прошлом месяце, то он должен автоматически подключиться к этим гугл таблицам, посчитать эту сумму и выдать пользователю ответ на этот вопрос.

Наш финансовый агент будет использовать mcp Google Sheets, для подключения и создания непосредственно финансовой таблички и занесение информации в нее. 

## Ресёрч-агент

Задача этого агента будет подключаться к MCP FireCrawl, а также Brave search - для того, чтобы искать информацию в интернете на определенный запрос, который мы будем ему задавать. его задача будет проанализировать детально ту тему, которую мы в него загрузили. Далее найти информацию в интернете по этой теме. собрать из разных источников эту информацию, после этого сформировать отчет пользователю в формате google docs (MCP)

## Ассистент
Твоя задача - мониторить мои встречи, мой календарь и моё расписание. Поэтому у тебя будет доступ к моему Google календарю с помощью MCP Google Calendar 
задача будет у тебя следующее:
1. ты должен подключаться к моему календарю, получать список задач на сегодняшний день и на неделю вперед. какие у меня задачи и встречи есть, анализировать моё свободное время и что конкретно у меня стоит на ближайшее время. 
2. по моему запросу выдавать мне те события и те мероприятия, которые у меня есть на тот день, который я запрошу 
3. вносить события, которые я запрошу в мой Google Календарь И проверять, есть ли у меня время на это событие, на тот день на который я захочу это все занести 
4. предлагать мне варианты оптимизации моего рабочего времени, Для того, чтобы если у меня есть большие окна, то мы могли переносить какие-то события, чтобы сжать мои временные затраты и освободить немного времени для других задач. либо же говорить мне о том, какие задачи я могу поставить, какие нет и когда конкретно эту задачу я могу по сути поставить. А также предлагать варианты по оптимизации моего рабочего времени для того, чтобы я мог, скажем так, меньше времени тратить, но при этом быть более эффективным с точки зрения тайм-менеджмента. 


Нам необходимо этих агентов создать в папке .claude/agents, так как мы будем запускать их в режиме субогентов, а также в режиме agent teams.
